# 0.76b_My_PuTTY vs 0.76_My_PuTTY 对比分析

## 概述

- **`0.76_My_PuTTY`**：基于 PuTTY 0.76 原始源码的 KiTTY 旧版，扁平目录结构，`ssh`/`win` 前缀命名
- **`0.76b_My_PuTTY`**：基于 PuTTY pre-0.77 开发版源码的 KiTTY 当前开发版，子目录模块化结构，通用命名
- **编译 KiTTY 使用的是 `0.76b_My_PuTTY`**（version_major.txt = `"999.76.1"`，BINARY_VERSION = `0,76,1,13`，git 工作区修改全部在此目录）

---

## 一、PuTTY 上游代码重构（0.76 → pre-0.77）

## 二、KiTTY 特有的新功能和增强

### 1. OSC52 剪贴板同步警告（新配置项）

- `putty.h`：新增 `CONF_osc52_warn_before_cliboard_sync` 配置项
- `settings.c`：新增 `OSC52WarnBeforeClipboardSync` 保存/加载
- `config.c`：在配置面板中显示此选项（仅非 mid-session 时）

### 2. 左右 Shift 键检测增强

- `window.c`：新增 `GetAsyncKeyState(VK_LSHIFT)` / `GetAsyncKeyState(VK_RSHIFT)` 检测，可区分左右 Shift 键

### 3. `make_title()` 动态内存分配重写

- 0.76 使用固定 1024 字节栈缓冲区
- 0.76b 改用 `malloc`/`realloc` 动态分配，消除了超长标题缓冲区溢出风险
- `set_title()` 和 `set_icon_title()` 同步更新，使用 `free()` 释放返回的指针

### 4. 连接关闭逻辑重构

- 0.76b 增加了 `is_backend_first_connected` 守卫条件，防止双重关闭或重入问题

### 5. `MOD_RUTTY` → `MOD_RUTTY1`

- WM_COPYDATA 处理器从 `#ifdef MOD_RUTTY` 改为 `#ifdef MOD_RUTTY1`，表示远程控制功能版本升级

### 6. 远程命令处理迁移

- `set_title()` 中的 `__` 前缀远程命令处理被注释掉，注释标明 `// moved in terminal.c`

### 7. 系统休眠后定时器修复

- `timing.c`：新增 `MOD_PERSO` 条件编译块，当第一个定时器严重超时（超过 10 秒）时也触发前端通知，解决 Windows 休眠/唤醒后定时器失效问题

### 8. KiTTY 日志头格式

- `logging.c`：新增 `MOD_PERSO` 块，将 PuTTY 标准日志头改为 KiTTY 专用格式：
  ```
  =~=~=~=~=~=~=~=~=~=~=~= KiTTY log ... =~=~=~=~=~=~=~=~=~=~=~=
  ```

### 9. `trayIcone` 声明修正

- `pageant.h`：从定义 `NOTIFYICONDATA trayIcone;` 改为 `extern NOTIFYICONDATA trayIcone;`，修复多重定义链接错误

### 10. WTS_VIRTUAL_CLASS 类型定义注释掉

- `window.c`：`typedef enum _WTS_VIRTUAL_CLASS` 被注释掉，因为新版 SDK 已提供该定义，避免重定义错误

## 四、链接器与构建系统变化

- 0.76b 对 `putty.exe` 使用 `-Wl,--start-group ... -Wl,--end-group` 循环链接，解决库依赖顺序问题
- 0.76b 为 `plink.exe`、`pscp.exe`、`psftp.exe` 新增链接 `libregex.a`
- 集成对象文件重命名：`winpgnt_integrated.o` → `winpageant_integrated.o`，`winpgen_integrated.o` → `winputtygen_integrated.o`
