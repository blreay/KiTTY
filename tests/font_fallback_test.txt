To exercise interactively:
  type 'cat tests/font_fallback_test.txt' inside a KiTTY session
  using Consolas as the primary font.

Acceptance:
  - All non-ASCII characters render glyphs (no .notdef boxes) when fallback fonts are installed.
  - U+23F5 ⏵ displays from Segoe UI Symbol fallback.
  - Cell alignment is preserved (no half-glyphs, no overlap).
  - With Log=info in [FontFallback], fontfallback.log records the per-codepoint slot mappings.


Known limitations (MVP, route A: GDI + GetGlyphIndicesW)
========================================================

1. Emoji renders monochrome at best. Non-BMP codepoints (U+10000 and
   above — most emoji, some CJK extensions, math alphanumerics) are
   NOT successfully fallback-mapped by the current GDI route, because
   GetGlyphIndicesW evaluates surrogate halves individually and they
   never map to a real glyph in any font. The cache records non-BMP
   codepoints as "all-miss" and they render via primary-font .notdef.

   Working around this requires Uniscribe (ScriptItemize+ScriptShape)
   or DirectWrite, both deferred to the second phase. For now, BMP
   characters (U+0000–U+FFFF — which includes Box Drawing, Block,
   Geometric Shapes, Misc Symbols, Dingbats, Arrows, CJK Unified
   Ideographs, Nerd Font PUA, and most everyday content) ARE
   successfully fallback-mapped.

   To work around the SMP limitation in the meantime, you may use:
   Override=1F600-1F64F:Segoe UI Emoji
   in [FontFallback] to force a slot assignment without probing.

2. Fallback glyph baseline may differ by 1-2 px from the primary font.
   No vertical adjustment is applied; the glyph is allowed to ride at
   its native ascent within the cell clip rectangle.

3. FONT_HIGH / FONT_WIDE (LATTR_TOP/BOT, double-height/width lines)
   variants are not created for fallback fonts. Characters that need
   fallback in doubled lines fall back to the primary font.

4. Multi-line overrides use a SINGLE Override= key with semicolon
   separators, not multiple Override= lines. Example:
   Override=E000-F8FF:Symbols Nerd Font Mono;1F600-1F64F:Segoe UI Emoji

5. RTL text mixed with fallback characters loses the GCP shaping that
   exact_textout would provide. Pure RTL text (no fallback needed)
   still uses the original path and is unaffected.

6. The BMP cache is a fixed 64 KB allocation. The SMP hash is capped
   at 1024 entries; codepoints beyond that limit re-probe each frame.

7. winfb_split caps at 64 runs per call. A pathological line that
   alternates between primary and fallback every cell beyond 64 cells
   will drop the tail. Realistic terminal lines do not hit this.

8. The fallback module never participates in cursor / underline /
   strikethrough / background-image painting decisions. Those continue
   to follow the original do_text_internal logic.

9. fontfallback.log defaults to OFF and lives next to the .exe.
   Rollover is single-generation at ~10 MB (.log -> .log.1).

10. font_varpitch (variable-pitch) terminal fonts: fallback runs always
    pass a non-NULL lpDx array to ExtTextOutW, which means GDI's
    TA_CENTER positioning by native advance is bypassed. For the
    overwhelmingly common monospace terminal case this has no effect.
    Variable-pitch + fallback users may see uncentered fallback glyphs.

11. After reset_window changes the primary font, the fallback subsystem
    is reinitialised twice: once by init_fonts directly, then again
    by the surrounding reset_window. The second pass is functionally
    a no-op but does redo the EnumFontFamiliesEx install-check and
    HFONT delete/recreate. Cost is < 10ms in practice. Future cleanup
    can collapse this.
